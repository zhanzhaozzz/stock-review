# Phase 3：前端页面切换与信息归位方案

## 文档目的

本阶段承接：

- Phase 0：统一语言
- Phase 1：核心对象建模
- Phase 2：生成链和调度链打通

Phase 3 的目标是：

**把前端从旧的“复盘工具箱”页面结构，切换为新的“三大主页面”结构。**

本阶段关注的是：

- 页面职责归位
- 页面区块重排
- 新对象接入
- 旧页面退位

本阶段不关注数据库迁移和旧表退役。

## 本阶段目标

### 要做

- 把首页切换为 `AI 作战台`
- 新建 `候选池` 页面
- 重构 `盘后复盘` 页面
- 调整侧边栏主入口
- 让主页面只消费 V1 四对象

### 不做

- 不删除旧页面文件
- 不删除旧 API
- 不做复杂视觉动效
- 不做移动端适配
- 不做多用户权限页面

## 页面职责回顾

### 1. AI 作战台

职责：定调

回答：

- 今天能不能做
- 今天适合什么风格
- 今日主叙事是什么
- 今日主要风险是什么

### 2. 候选池

职责：聚焦

回答：

- 今天到底盯谁
- 为什么盯
- 能不能碰
- 怎么碰

### 3. 盘后复盘

职责：校正

回答：

- 早盘判断准不准
- 候选池哪些成立
- 明天该带什么

## 首页改造：AI 作战台

### 目标文件

- 以现有 `frontend/src/pages/Dashboard.tsx` 为基础重构

### 页面顺序

1. 全局定调区
2. 纪律警报区
3. 市场状态与叙事区
4. 候选池预览区

### 数据来源

- `MarketStateDaily`
- `BattleBrief`
- `CandidatePoolEntry` 的预览结果

### 推荐接口

- `GET /api/v1/combat-desk/today`

### 页面块定义

#### Block 1：全局定调区

必须显示：

- 今日状态
- 建议仓位
- 市场阶段
- 风格标签
- 一句话总结

#### Block 2：纪律警报区

必须显示：

- 禁止动作
- 风险红线

要求：

- 最多展示当天最重要的 1 到 3 条
- 不做警报疲劳

#### Block 3：市场状态与叙事区

布局建议：

- 左栏：客观市场状态
- 右栏：今日叙事

左栏显示：

- 情绪分
- 涨跌停
- 炸板率
- 最高连板
- 成交额

右栏显示：

- 宏观链路
- 今日主叙事
- 偏多方向
- 偏空方向
- 风险提示

#### Block 4：候选池预览区

只显示：

- Top 5 到 Top 10
- `通过` 和 `观察` 状态为主
- 每个候选至少保留：
  - 名称
  - 主题
  - 一句逻辑
  - 门控状态

### 首页禁止混入的内容

- 全市场长表格
- 新闻流全文
- 大段 Markdown 长文
- 复杂个股技术图
- 盘后验证内容

## 候选池页面

### 目标文件

- 新建 `frontend/src/pages/CandidatePool.tsx`

### 页面目标

把当天真正值得盯的少量标的，按安检逻辑分层展示。

### 页面顺序

1. 状态过滤器
2. 候选卡片流
3. 详情抽屉或详情面板

### 数据来源

- `CandidatePoolEntry`

### 推荐接口

- `GET /api/v1/candidates/today`
- `PUT /api/v1/candidates/{id}`

### 页面块定义

#### Block 1：状态过滤器

显示：

- `通过`
- `观察`
- `拦截`

每项带计数。

默认：

- 勾选 `通过`
- 勾选 `观察`
- 不默认显示 `拦截`

#### Block 2：候选卡片流

每张卡必须至少展示：

- 代码 + 名称
- 来源类型
- 主题标签
- 核心逻辑
- 风险标签
- 门控状态
- 门控理由
- 操作倾向

#### Block 3：卡片交互

V1 可支持的轻交互：

- 标记带到明天
- 标记忽略
- 查看详情
- 回填盘后验证结果

### 候选池页面禁止混入的内容

- 长期自选股全集
- 全市场涨幅排行
- 与当天无关的分析历史
- 盘后全局复盘

## 盘后复盘页面

### 目标文件

- 在现有 `frontend/src/pages/DailyReview.tsx` 基础上重构，或逐步替换为新的盘后复盘页

### 页面目标

对当天早盘判断和候选池执行效果进行校正，并为第二天输出承接。

### 数据来源

- `BattleBrief`
- `MarketStateDaily`
- `CandidatePoolEntry`
- `PostMarketReview`

### 推荐接口

- `GET /api/v1/post-market-review/today`
- `GET /api/v1/candidates/today`

### 页面顺序

1. 今日判断回放
2. 市场结果复盘
3. 候选池验证
4. 明日承接

### 页面块定义

#### Block 1：今日判断回放

显示：

- 早盘定调
- 早盘主叙事
- 早盘风险提示
- 今日打分结果

#### Block 2：市场结果复盘

显示：

- 实际市场走向
- 主线是否兑现
- 风格是否变化

#### Block 3：候选池验证

逐票显示：

- `逻辑兑现`
- `时机未到`
- `逻辑证伪`
- `纪律拦截正确`

以及一句归因说明。

#### Block 4：明日承接

显示：

- 带走的主题
- 明日种子
- 剔除的方向

### 盘后复盘页面禁止混入的内容

- 重新写一篇长文复盘
- 纯情绪抒发
- 与第二天无关的总结

## 侧边栏调整方案

### 旧结构问题

当前侧边栏更偏：

- 市场总览
- 自选股
- 每日复盘
- 历史复盘

这更像工具菜单，不是作战流菜单。

### 新结构建议

主入口只保留：

- `AI 作战台`
- `候选池`
- `盘后复盘`
- `自选股`
- `设置`

说明：

- `自选股` 继续保留
- `新闻 / 评级 / 战法 / 操作记录` 可以保留路由，但不再做主入口

## 前端接入顺序

### Step 1

先完成 `AI 作战台` 首页接入

### Step 2

再新建 `候选池` 页面

### Step 3

再重构 `盘后复盘`

### Step 4

最后调整侧边栏和默认入口

## 本阶段验证标准

完成后，必须满足：

- 首页第一屏能清楚给出今日状态和风格
- 候选池页面能只围绕 `CandidatePoolEntry` 工作
- 盘后复盘页面能显示早盘判断 vs 盘后结果
- 页面之间职责不再混乱
- 旧工具页仍可暂时保留，但不再是主工作流

## 当前阶段禁止动作

- 不删旧页面
- 不删旧接口
- 不做移动端
- 不做复杂图表堆砌
- 不把候选池做成全市场排行页

## 最终结论

Phase 3 的任务不是“把页面做漂亮”，而是：

**把前端页面的职责彻底归位，让 V1 三大页面真正对应“定调 -> 聚焦 -> 校正”的日常决策流。**
